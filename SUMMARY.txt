🔄 OFFLINE SYNC WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User creates farmer data (offline)
   ↓
2. Data saved to SQLite (syncStatus = 'PENDING')
   ↓
3. Background task checks internet (every 15 min)
   ↓
4. If online → Send pending data to server
   ↓
5. Server validates & saves to MongoDB
   ↓
6. Server returns updated data
   ↓
7. App updates local DB (syncStatus = 'SYNCED')
   ↓
8. Conflicts resolved using LATEST_WINS strategy
